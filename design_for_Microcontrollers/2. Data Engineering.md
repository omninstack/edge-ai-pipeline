The Data Engineering layer is responsible for transforming raw industrial field data into AI-ready datasets for training, validation, deployment, and continuous improvement of Edge AI models.

This layer is critical for:
* **Predictive Maintenance:** Early detection of equipment failure.
* **Fault Detection:** Identifying operational anomalies in real-time.
* **False Alarm Reduction:** Using multi-sensor fusion to validate events.
* **Digital Twin Synchronization:** Maintaining accurate virtual representations.
* **Autonomous Industrial Analytics:** Enabling edge-side decision making.

---

# 2.1 Objectives

## Primary Goals
* **Reliable Collection:** Ensure high-fidelity industrial telemetry.
* **Normalization:** Standardize heterogeneous protocol data (Modbus, BACnet, etc.).
* **Supervised Learning Support:** Build high-quality labeled datasets.
* **Lineage & Traceability:** Maintain strict dataset versioning and audit trails.
* **CI/CD for AI:** Enable automated retraining and deployment pipelines.
* **Hybrid Workflows:** Seamlessly support both Edge and Cloud processing.

---

# 2.2 Data Sources

## Industrial Device Inputs
* **Environmental & Process Sensors:** Temperature, Humidity, Pressure, Smoke, Gas, Flow rate.
* **Electromechanical Sensors:** Vibration (Accelerometers), Current, Voltage.
* **Industrial Controllers (PLCs):** Rockwell, Siemens, Mitsubishi, Omron, etc.

## Supported Protocols
* **Serial/Fieldbus:** Modbus RTU, BACnet MS/TP, CAN bus.
* **Ethernet-Based:** Modbus TCP, BACnet/IP, EtherNet/IP, OPC UA.

# 2.4 Data Collection & Ingestion

## Edge-Side Responsibilities
* **Device Polling:** Real-time acquisition from industrial registers.
* **Protocol Normalization:** Mapping vendor-specific data to a unified schema.
* **Precise Timestamping:** Ensuring millisecond-accurate event ordering.
* **Local Buffering:** Resilience against temporary network outages.

### Standardized Telemetry Payload (Example)
```json
{
  "device_metadata": {
    "id": "motor_01",
    "type": "induction_motor",
    "location": "line_3"
  },
  "telemetry": {
    "timestamp": "2026-05-14T10:30:00.005Z",
    "temperature_c": 78.2,
    "vibration_rms": 0.42,
    "current_a": 14.8
  },
  "status": "nominal"
}
```

## MQTT Messaging Strategy
Data is routed through a hierarchical topic structure for efficient subscription management:
* `factory/site/line/device/telemetry` — Continuous data stream.
* `factory/site/line/device/events` — Alarms and state changes.
* `factory/site/line/device/inference` — Edge-side AI results.

# 2.5 Annotation & Labeling

Annotation transforms raw telemetry into high-value training data for supervised learning.

## Annotation Taxonomy
| Type | Industrial Context | Example |
| :--- | :--- | :--- |
| **Classification** | State Identification | Normal vs. Bearing Fault |
| **Detection** | Event Identification | Fire/Gas leak detected |
| **Regression** | Health Estimation | Remaining Useful Life (RUL) in hours |
| **Segmentation** | Temporal Partitioning | Identifying "Startup", "Steady State", "Shutdown" phases |

## Labeling Sources
* **Expert Manual Labels:** Input from maintenance engineers and operators.
* **Automated Logic Labels:** Derived from PLC error codes and hard thresholds.
* **Historical Records:** Correlating data with past maintenance logs.

# 2.6 Dataset Management & Versioning

Ensuring reproducibility and auditability of AI models through strict data lineage.

## Versioning Strategy
* **Data Diffs:** Tracking changes in labeling and preprocessing logic.
* **Feature Store:** Centralized repository for engineered features.
* **Train/Test Splits:** Immutable records of dataset partitions used for training.

## Tooling Ecosystem
* **DVC (Data Version Control):** Managing large datasets alongside code.
* **MLflow:** Tracking experiments and dataset-to-model relationships.
* **MinIO / S3:** Scalable object storage for raw and processed data.

---

# 2.7 Data Preprocessing & Feature Engineering

Transforming noisy industrial signals into input features for AI models.

## Signal Processing
* **Cleaning:** Outlier detection and noise filtering (e.g., Kalman filters).
* **Time Sync:** Aligning data from asynchronous sources (PLCs vs. direct sensors).
* **Normalization:** Scaling features for neural network stability (Min-Max, Z-Score).

## Feature Extraction
* **Time-Domain:** Mean, RMS, Peak-to-Peak, Kurtosis, Skewness.
* **Frequency-Domain:** FFT (Fast Fourier Transform), Spectrograms, Wavelet Analysis.
* **Temporal Windowing:** Sliding windows with configurable overlap (e.g., 5s window, 50% overlap).

# 2.10 Security & Data Integrity

* **Transport Security:** TLS 1.2/1.3 for all MQTT and API traffic.
* **Integrity:** Checksums and digital signatures for dataset files.
* **Access Control:** Role-Based Access Control (RBAC) for data repositories.
* **Anonymization:** Removing sensitive operator identifiers where applicable.
